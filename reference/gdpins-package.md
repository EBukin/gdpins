# gdpins: Google Drive + pins Data Management for R Projects

Layers three storage models over Google Drive: versioned pins boards for
processed data (data-raw, data-interm, data-clean, output-tables), a
plain-file raw-exogenous connection for API/source data, and a
local-first output layer. All Drive access is lazy-authenticated,
offline-aware, sync-explicit, and delete-guarded.
